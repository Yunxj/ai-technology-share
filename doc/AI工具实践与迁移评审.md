## 一、AI工具使用最佳实践

系统梳理 AI 辅助开发中的上下文管理、插件应用、文档处理与多模态生成等核心策略与优化方案。

### 1. 上下文与 TOKEN 管理（效率关键）

- 保持 Prompt 简洁，避免上下文过长导致响应缓慢
- 手动提取关键错误信息，避免提交全文或大段日志
- 用户级与项目级云端配置应分别存储通用规范与项目结构
- 安装插件实时查看 TOKEN 余额，避免过量消耗

### 2. 插件与功能扩展（效率工具）

- 推荐安装 SUPERPOWERS、头脑风暴等插件提升编码与思考效率
- 使用 code reviewer 类 Agent 进行代码质量检查
- 通过 spec kit 辅助生成接口调用逻辑
- Dashboard 类工具可快速搭建前端原型，尤其适合后端人员

### 3. 文档与接口处理策略（稳定优先）

- 手动下载官方 Vue/React/Spring 文档为 Markdown 或 TXT 后上传解析
- 避免依赖 MCP 网络搜索（每月仅 100 次调用）
- 钉钉等平台接口优先使用内置 MCP 或本地文档导入方式

### 4. 前端 UI 生成挑战（还原瓶颈）

- 复杂 C 端场景（如小程序）从图片直接生成前端代码还原度较低
- 智谱多模态能力偏弱，建议尝试通义 3.5 plus、MiniMax 2.5、Kimi 2.5
- 设计稿可先上传豆包或通义千问生成高质量 Prompt，再用于编码
- MarsGo/Figma 等工具导出内容若不规范，会影响 AI 理解

### 5. 生成效果提升建议（现实预期）

- AI 目前能完成 60%～80% 工作量，但细节样式仍需人工调整
- GPT-4 移动端生成效果比国产模型好 20%～30%，PC 端还原度可达 60%～70%
- 团队可共享高端模型 TOKEN 测试，或个人花 7.9 元购买通义服务验证

### 6. 全局提示词配置（复用机制）

- 将封装规范、编码要求等写入用户级 `.json` 配置文件，跨项目复用
- 在 Prompt 中用加粗标注「非常重要」等字段，增强 AI 注意力权重
- 全局上下文每次克隆环境启动时自动加载，无需重复设置

## 二、前端开发中的 AI 应用挑战

### 1. 基于 UI 设计稿生成页面

- 从图片直接生成前端代码在复杂 C 端场景和小程序中常出现还原度问题
- 智谱多模态能力偏弱，建议尝试通义 3.5 plus、MiniMax 2.5、Kimi 2.5
- 可将设计稿先上传豆包或通义千问生成高质量 Prompt，再用于 AI 编码
- MarsGo/Figma 等原型工具导出若不规范，会降低 AI 理解准确度

### 2. 提升生成效果的建议

- AI 可完成约 60%～80% 工作量，但细节样式仍需人工调整
- GPT-4 移动端生成效果比国产模型好 20%～30%，PC 端可达 60%～70% 还原度
- 团队可共享高端模型 TOKEN 测试，或个人购买通义服务（7.9 元）验证
