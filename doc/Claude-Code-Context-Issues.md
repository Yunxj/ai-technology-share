# Claude Code 上下文问题说明

## 常见问题

### 1. 会话结束即遗忘

新会话启动时，Claude Code 无法自动延续上一会话的上下文。项目背景、需求细节、调试历程等需要人工重新交代，导致重复沟通、效率下降。

### 2. 长对话 Token 超限

当对话轮次或涉及文件较多时，有效上下文容易被截断。早期信息（如技术方案、约束条件）可能被挤出窗口，导致后续回复偏离初衷或遗漏关键约束。

### 3. 多轮调试、迭代时信息易丢失

在 Bug 排查、功能迭代等场景中，问题现象、复现步骤、已尝试方案等分散在多轮对话中，难以被稳定保留和检索，影响问题定位和决策连贯性。

## 推荐解决方案

- **Claude-Mem**：开源持久化记忆插件，自动捕获、压缩、存储会话上下文，新会话智能注入相关记忆，可显著缓解「AI 失忆」问题。
- **Superpowers**：工作流与技能库，通过系统化流程（头脑风暴 → 计划 → 子代理执行）和 TDD/调试等技能，减少对长上下文的依赖，提升任务执行的连贯性。
