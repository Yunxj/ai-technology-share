# 需求开发先规划：功能拆分与 Cursor Plan 模式

---

## 一、需求开发前：功能拆分与思路理清

### 1. 将大功能拆成多个小功能

复杂需求应先拆解为可独立完成的小任务：

- 降低单次改动的复杂度
- 便于分步验证与回滚
- AI 更容易理解并执行

### 2. 开发前理清思路

- 明确目标：这个需求最终要达成什么效果？
- 梳理依赖：哪些模块/文件会受影响？
- 预估风险：可能引入的副作用、测试重点

---

## 二、Cursor Plan 模式工作流程

Plan 模式遵循「先规划后执行」，让 AI 像项目经理一样管理复杂开发任务。

### 步骤 1：自动调研 (Research)

提出需求后，AI 会：

- 自动扫描代码库，查找相关文件和文档
- 向你提出澄清问题，以完全理解意图

### 步骤 2：生成计划 (Plan)

AI 生成 Markdown 格式的详细计划，包含：

- 具体实施步骤
- 涉及的文件与模块
- 技术选型与潜在风险

### 步骤 3：审阅与编辑

你可以直接修改计划：

- 调整步骤顺序
- 添加或删除待办事项
- 保存为工作区中的 Markdown 文件供日后参考

### 步骤 4：Build 执行

确认计划无误后，点击 **Build**，Cursor 严格按照计划逐步执行代码生成。

---

## 三、适用场景

Plan 模式特别适合：

| 场景 | 说明 |
|------|------|
| 代码重构 | 多文件联动、依赖关系梳理 |
| 新功能模块 | 如用户权限系统、支付流程 |
| 跨文件改造 | 页面结构调整、组件抽离 |
| 团队协作 | 大型功能开发，多人配合 |

---

## 四、使用技巧

1. **不要盲目 Build**：认真审核计划，利用可编辑特性修正 AI 理解偏差
2. **明确需求描述**：进入 Plan 模式前清晰描述功能目标、技术要求、预期效果
3. **组合使用**：复杂任务先用 Plan 搭框架，细节调整再用 Chat 或 Agent 模式
4. **快捷开启**：在 Agent 输入框按 **Shift + Tab** 切换至 Plan 模式

---

## 五、Plan 模式 vs Agent 模式

| 特性 | Plan 模式 | Agent 模式 |
|------|-----------|------------|
| 工作方式 | 先规划后执行 | 直接执行 |
| 适用任务 | 复杂、多步骤、多文件 | 简单、快速、单点修改 |
| 优势 | 稳定、可控、易审查 | 快速迭代 |
