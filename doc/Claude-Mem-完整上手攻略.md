# Claude-Mem 完整上手攻略

## 一、简介

Claude-Mem 是开源的 Claude Code 持久化记忆插件，专门解决 AI 助手的「失忆」问题。它能自动捕获开发会话中的信息，进行语义压缩后存储，并在新会话中智能注入相关上下文，实现跨会话的知识连续性。

- **项目**：<https://github.com/thedotmack/claude-mem>
- **协议**：AGPL 3.0
- **技术**：TypeScript、Node.js 18+、SQLite + FTS5、Chroma、Bun

---

## 二、核心能力

### 自动记忆持久化

- 无需手动干预，自动捕获上下文信息：项目需求、代码片段、调试对话等
- 压缩后存储到本地数据库，重启会话自动加载历史记忆

### 渐进式记忆披露

- 分层级、分优先级注入记忆内容
- 仅加载轻量级「索引」，需要时才获取完整内容
- 平均可节省约 2,250 个 Token

### 智能搜索

- 支持自然语言搜索项目历史
- 基于 FTS5 全文搜索和向量搜索的混合架构
- 可直接问 Claude：「这个 Bug 之前修过吗？」

### Web 可视化界面

- 访问 `http://localhost:37777` 查看实时记忆流
- 可进行记忆筛选、删除等操作

### 隐私控制

- 用 ` ` 标签包裹敏感内容可排除存储
- 完全掌控哪些信息被持久化

---

## 三、安装

在 Claude Code 终端中执行：

```bash
/plugin marketplace add thedotmack/claude-mem
/plugin install claude-mem
```

重启 Claude Code 后，之前会话的上下文会自动出现在新会话中。

## 四、技术架构

### 核心组件

| 技术模块 | 选型 |
|---------|------|
| 开发语言 | TypeScript |
| 运行环境 | Node.js 18.0.0+ |
| 数据库 | SQLite + FTS5（结构化）+ Chroma（向量化）|
| Web 服务 | Bun |

### 四大生命周期钩子

- **SessionStart**：新会话启动时注入上下文
- **UserPromptSubmit**：捕获用户请求
- **PostToolUse**：记录工具执行（代码编辑、文件操作、Shell 命令等）
- **Stop**：会话结束时生成摘要

### 工作流程

```
会话开始 → 注入历史上下文 → 捕获用户提示和工具执行 → Worker 处理学习内容 → 会话结束时生成 AI 驱动摘要 → 为下次会话做准备
```

---

## 五、典型场景

Claude-Mem 特别适合：

- **长期迭代的项目开发**：持续保留架构设计和迭代思路
- **复杂代码调试**：记录问题与解决方案
- **团队协作开发**：新成员快速接手项目背景
- **多设备协同开发**：跨设备同步上下文

与手动维护记录相比，Claude-Mem 自动完成捕获、压缩、存储全过程，大幅降低 Token 消耗和人工成本。
